# MCP RAQ 1C

MCP-сервер с RAG для метаданных 1С:Предприятие. Позволяет ИИ-агентам (Claude Code, Cursor, VS Code Copilot, RooCode) получать точную информацию о структуре конкретной конфигурации 1С — вместо того чтобы галлюцинировать имена объектов и реквизитов.

## Зачем это нужно

Когда ИИ-агент пишет код или запросы для 1С, он не знает структуру вашей конфигурации: какие справочники существуют, как называются реквизиты, какие типы у полей. Особенно это критично для редких отраслевых конфигураций, структура которых неизвестна стандартным LLM.

**MCP RAQ 1C** решает эту проблему: агент может в реальном времени запросить метаданные конфигурации через MCP-протокол и получить точные имена объектов, реквизитов и их типы.

### Пример использования

Вы спрашиваете агента:
> Напиши запрос, который выводит все заказы клиентов за сегодня с суммой больше 100 000

Агент:
1. Вызывает `1c_metadata_search("заказ клиента")` → находит документ `SS_ЗаказКлиента`
2. Вызывает `1c_metadata_details("SS_ЗаказКлиента")` → получает реквизиты: `СуммаДокумента (Число(15,2))`, `Статус`, табличную часть `Состав`
3. Генерирует корректный запрос с **реальными** именами объектов и полей

## Поддерживаемые конфигурации

Система работает с **любой** конфигурацией 1С:Предприятие 8.3, из которой выполнена стандартная выгрузка в XML (Конфигуратор → «Выгрузить конфигурацию в файлы»). Протестировано на конфигурациях с ~6000 объектов метаданных.

### Микросервисы

| Сервис | Технологии | Назначение |
|--------|-----------|------------|
| **parser** | Python, FastAPI | Парсинг XML-выгрузки конфигуратора 1С → структурированные описания (имя, синоним, реквизиты, типы, табличные части, движения регистров) |
| **embeddings** | Python, FastAPI, BERTA, fastembed | Генерация dense-векторов (768 измерений) + sparse-векторов (BM25) для гибридного поиска |
| **loader** | Python, Streamlit | Веб-интерфейс для индексации: запускает парсинг → эмбеддинги → загрузку в Qdrant |
| **mcp-server** | TypeScript, Express, MCP SDK | MCP-протокол (SSE + Streamable HTTP) — предоставляет инструменты поиска для ИИ-агентов |
| **qdrant** | Qdrant | Векторная БД: named vectors (object_name, friendly_name) + sparse (bm25), RRF fusion |

### Как работает поиск

1. Запрос от агента (например, «остатки товаров на складе») преобразуется в dense- и sparse-векторы
2. В Qdrant выполняется три параллельных поиска:
   - По вектору `object_name` (техническое имя) — dense
   - По вектору `friendly_name` (синоним) — dense
   - По вектору `bm25` (точное текстовое совпадение) — sparse
3. Результаты объединяются через **RRF (Reciprocal Rank Fusion)** и ранжируются
4. Агент получает компактный список: имя, синоним, тип, score

## Быстрый старт

### Предварительные требования

- Docker Desktop (8 ГБ RAM, 4 ядра)
- Выгрузка конфигурации 1С в XML (стандартная «Выгрузить конфигурацию в файлы» из конфигуратора) — путь указывается в настройках Loader

### Запуск

```bash
cp .env.example .env
docker compose up -d
```

### Индексация метаданных

1. Откройте Loader UI: http://localhost:8501
2. Убедитесь, что все сервисы зелёные (Parser, Embeddings, Qdrant)
3. Нажмите **«Индексировать»**
4. Дождитесь завершения (занимает несколько минут в зависимости от размера конфигурации)

### Порты

| Сервис | Порт | Назначение |
|--------|------|------------|
| MCP Server | 8000 | MCP-протокол (SSE + Streamable HTTP) |
| Loader | 8501 | Streamlit UI для индексации |
| Parser | 8001 | API парсинга XML |
| Embeddings | 5050 | API эмбеддингов |
| Qdrant | 6333 | Векторная БД |
| Qdrant Dashboard | 6334 | Веб-интерфейс Qdrant |

## MCP Tools

### `1c_metadata_search`

Семантический поиск объектов метаданных. Находит объекты по смыслу, а не только по точному совпадению имени.

**Параметры:**
- `query` (string) — поисковый запрос на русском или английском
- `objectType` (string, опционально) — фильтр по типу: `Catalog`, `Document`, `InformationRegister` и т.д.
- `limit` (number, по умолчанию 10) — количество результатов

**Пример ответа:**
```
1. SS_ЗаказКлиента | Заказ клиента | Документ | score: 1.00
2. Резерв | Заказ покупателя | Документ | score: 0.33
```

### `1c_metadata_details`

Полное описание объекта: реквизиты с типами, табличные части, движения регистров.

**Параметры:**
- `name` (string) — техническое имя объекта (из результатов поиска)
- `objectType` (string, опционально) — тип объекта (уточняет при совпадении имён)

**Пример ответа:**
```
Объект: SS_ЗаказКлиента
Тип: Документ
Синоним: Заказ клиента

Реквизиты:
- Фирма (СправочникСсылка.Организации)
- СуммаДокумента (Число(15,2))
- Статус (СправочникСсылка.SS_СтатусыЗаказовКлиентов)

Табличные части:
- Состав (Номенклатура, Количество, Цена, Сумма, ...)

Движения регистров:
- AccumulationRegister.SS_УчетЗаказовКлиентов
```

### `1c_metadata_types`

Статистика по типам объектов в коллекции. Полезно для обзора конфигурации.

## Подключение к IDE

### Claude Code

Добавьте в `~/.claude.json` (секция `mcpServers`):
```json
{
  "1c-metadata": {
    "type": "sse",
    "url": "http://localhost:8000/sse"
  }
}
```

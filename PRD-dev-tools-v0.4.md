## 1. Обзор

Расширение MCP-сервера 4 инструментами для разработчиков 1С: проверка синтаксиса BSL, справка по платформе, справка по БСП, шаблоны кода. Заменяет проприетарные comol-серверы (невалидные лицензии) собственными решениями на существующей инфраструктуре (Qdrant + BERTA + MCP Server).

**Результат:** MCP-сервер с 13 инструментами — полный набор для аналитика и разработчика 1С.

## 2. Проблема

- 4 comol-сервера не работают — лицензионные ключи невалидны, зависимость от вендора
- Каждый comol-сервер — отдельный контейнер с ChromaDB + своя embedding-модель = ~10 ГБ × 4
- Разработчик 1С не может проверить синтаксис, найти справку или шаблон через MCP
- Стоимость бездействия: разработчики пишут код без контекста платформы → ошибки, переделки

## 3. Цели и метрики

- **North Star:** 13 рабочих MCP-инструментов (9 текущих + 4 новых)
- **KPI:**
  - `1c_syntax_check`: время ответа < 5 сек для фрагмента до 1000 строк
  - `1c_help_search`: релевантный результат в топ-3 для 80% запросов
  - `1c_bsp_search`: аналогично help
  - `1c_templates`: находит шаблон по описанию задачи

## 4. Целевая аудитория

- **Разработчик 1С** — пишет код в IDE с MCP-агентом, хочет проверить синтаксис и найти справку
- **Аналитик 1С** — хочет найти описание механизмов БСП для понимания бизнес-логики

User Stories:
- Как разработчик, я хочу проверить BSL-код на синтаксические ошибки прямо в чате, чтобы не переключаться в конфигуратор
- Как разработчик, я хочу найти описание метода платформы по названию, чтобы понять его параметры и поведение
- Как разработчик, я хочу получить шаблон кода для типовой задачи (обработка проведения, подписка на событие), чтобы не писать с нуля
- Как аналитик, я хочу найти описание подсистемы БСП, чтобы понять какой механизм использовать

## 5. Scope: MVP

### Входит в MVP:
1. `1c_syntax_check` — BSL Language Server как sidecar, MCP-инструмент для проверки кода
2. `1c_help_search` — парсинг HBK-файла справки платформы → Qdrant → поиск
3. `1c_bsp_search` — индексация 961 HTML-файла справки БСП → Qdrant → поиск
4. `1c_templates` — коллекция шаблонов кода 1С → Qdrant → поиск
5. Loader: 3 новые вкладки (Справка платформы, Справка БСП, Шаблоны)
6. Документация: обновить agents.md, tools-map.md, ide-setup.md

### НЕ входит в MVP:
- Веб-редактор шаблонов (как у comol template-search-mcp)
- Автоматическая проверка синтаксиса при каждом `1c_code_search`
- Интеграция с EDT (Eclipse)
- Кастомные правила BSL LS (используем дефолтные)
- Автоматический сбор шаблонов с Infostart/GitHub

## 6. Функциональные требования

### 6.1. 1c_syntax_check — Проверка синтаксиса BSL

**Описание:** Принимает BSL-код, проверяет через BSL Language Server, возвращает диагностики.

**Поведение:**
1. Пользователь/агент вызывает `1c_syntax_check(code: "...")`
2. MCP-сервер отправляет код в sidecar-контейнер bsl-ls
3. BSL LS анализирует код и возвращает список диагностик
4. MCP-сервер форматирует результат в TSV

**Входные данные:**
- `code` (string, обязательный) — BSL-код для проверки
- `diagnostics_filter` (string, опциональный) — фильтр по severity: "error" | "warning" | "info" | "all" (default: "all")

**Выходные данные (TSV):**
```
Строка	Severity	Код	Сообщение
5	Error	S:1002	Процедура не найдена: "НесуществующаяФункция"
12	Warning	S:1028	Неиспользуемая переменная "Х"
```

Если ошибок нет: `Синтаксических ошибок не найдено (проверено N строк)`

**Граничные случаи:**
- Пустой код → `Ошибка: код не предоставлен`
- Код > 10 000 строк → обрезать с предупреждением
- BSL LS недоступен → `Ошибка: сервис проверки синтаксиса недоступен`

**Acceptance Criteria:**
- GIVEN валидный BSL-код WHEN вызов 1c_syntax_check THEN ответ "ошибок не найдено"
- GIVEN BSL-код с опечаткой WHEN вызов 1c_syntax_check THEN ответ содержит строку и описание ошибки
- GIVEN BSL LS контейнер остановлен WHEN вызов 1c_syntax_check THEN ответ с ошибкой недоступности

### 6.2. 1c_help_search — Справка по платформе 1С

**Описание:** Семантический поиск по справке платформы 1С:Предприятие 8.3.

**Поведение:**
1. Одноразовая индексация: HBK-файл (`bin/1cv8_ru.hbk`) → парсинг → чанки → BERTA embeddings → Qdrant коллекция `help_1c`
2. При запросе: гибридный поиск (dense BERTA + sparse BM25 + RRF) по коллекции
3. Возврат: заголовок статьи, релевантный фрагмент, путь в справке

**Входные данные:**
- `query` (string, обязательный) — поисковый запрос (например: "СоздатьОбъект", "ПолучитьФорму", "ОбщийМодуль")
- `limit` (number, по умолчанию 5, макс 20)

**Выходные данные (TSV):**
```
Заголовок	Раздел	Фрагмент	Score
СоздатьОбъект	Работа с объектами	Создаёт новый объект указанного типа...	0.87
```

**Граничные случаи:**
- Коллекция не проиндексирована → `Ошибка: справка платформы не проиндексирована. Запустите индексацию через Loader`
- Запрос не найден → `По запросу "..." ничего не найдено. Попробуйте другие ключевые слова`

**Acceptance Criteria:**
- GIVEN проиндексированная справка WHEN поиск "ТекущаяДата" THEN в результатах есть статья о функции ТекущаяДата()
- GIVEN непроиндексированная коллекция WHEN поиск THEN информативное сообщение об ошибке

### 6.3. 1c_bsp_search — Справка по БСП

**Описание:** Семантический поиск по справке Библиотеки стандартных подсистем (БСП).

**Поведение:**
1. Одноразовая индексация: 961 HTML-файл из `configuration/Prod/**/Help/ru.html` → чанки → BERTA → Qdrant коллекция `bsp_1c`
2. Поиск аналогично `1c_help_search`

**Входные данные:**
- `query` (string, обязательный) — описание или название подсистемы/механизма БСП
- `limit` (number, по умолчанию 5, макс 20)

**Выходные данные (TSV):**
```
Подсистема	Заголовок	Фрагмент	Score
УправлениеДоступом	Ограничение доступа	Механизм ограничения прав на уровне записей...	0.92
```

**Граничные случаи:** аналогично 6.2

**Acceptance Criteria:**
- GIVEN проиндексированная справка WHEN поиск "управление доступом" THEN результат содержит описание подсистемы УправлениеДоступом
- GIVEN проиндексированная справка WHEN поиск "контактная информация" THEN результат содержит справку подсистемы КонтактнаяИнформация

### 6.4. 1c_templates — Шаблоны кода 1С

**Описание:** Поиск шаблонов/сниппетов BSL-кода по описанию задачи.

**Поведение:**
1. Одноразовая индексация: JSON-файл с шаблонами → BERTA → Qdrant коллекция `templates_1c`
2. Каждый шаблон: название, описание, теги, BSL-код, контекст использования
3. Поиск: по описанию задачи находит подходящий шаблон с кодом

**Формат шаблона (JSON):**
```json
{
  "name": "ОбработкаПроведения",
  "description": "Шаблон процедуры обработки проведения документа с движениями по регистру накопления",
  "tags": ["проведение", "документ", "регистр", "движения"],
  "context": "Модуль объекта документа",
  "code": "Процедура ОбработкаПроведения(Отказ, РежимПроведения)\n\t// Движения по регистру\n\tДвижения.ИмяРегистра.Записывать = Истина;\n\t...\nКонецПроцедуры"
}
```

**Входные данные:**
- `query` (string, обязательный) — описание задачи (например: "обработка проведения документа")
- `tags` (string, опциональный) — фильтр по тегам через запятую
- `limit` (number, по умолчанию 3, макс 10)

**Выходные данные:**
```
## ОбработкаПроведения
**Контекст:** Модуль объекта документа
**Теги:** проведение, документ, регистр, движения

​```bsl
Процедура ОбработкаПроведения(Отказ, РежимПроведения)
    ...
КонецПроцедуры
​```
```

**Acceptance Criteria:**
- GIVEN коллекция с шаблонами WHEN поиск "подписка на событие" THEN возвращает шаблон с кодом подписки
- GIVEN пустая коллекция WHEN поиск THEN информативное сообщение

## 10. Технический стек

Всё встраивается в существующую инфраструктуру:

| Компонент | Технология | Изменения |
|-----------|-----------|-----------|

| Embeddings | BERTA | Без изменений (используется существующий) |

### BSL LS sidecar

```dockerfile
FROM eclipse-temurin:17-jre-alpine
RUN apk add --no-cache python3 py3-pip py3-flask
COPY bsl-language-server-0.28.5-exec.jar /app/bsl-ls.jar
COPY server.py /app/server.py
EXPOSE 8005
CMD ["python3", "/app/server.py"]
```

`server.py` — Flask-обёртка (~50 строк):
1. Принимает POST `/analyze` с JSON `{code, diagnostics_filter}`
2. Записывает код во временный `.bsl` файл
3. Вызывает `java -jar bsl-ls.jar --analyze --reporter json <tmpfile>`
4. Парсит JSON-отчёт, возвращает диагностики

### HBK-парсер

Формат HBK — zip-архив с HTML-файлами внутри. Парсинг:
1. Разархивировать `1cv8_ru.hbk` (это zip)
2. Извлечь HTML-файлы
3. BeautifulSoup: title → заголовок, text → контент
4. Чанкинг: по 500 токенов с перекрытием 100
5. Отправить в Embeddings → Qdrant

### Шаблоны кода

Начальная коллекция (~50-100 шаблонов) в `templates/1c-templates.json`:
- Обработка проведения документа
- Подписка на событие
- Обработка заполнения
- Печатная форма
- Фоновое задание
- Запрос к регистру
- Работа с таблицей значений
- HTTP-запрос из 1С
- Чтение/запись XML
- Работа с файлами
- Типовые обработки БСП
- и др.

## 13. Риски и открытые вопросы

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| HBK-формат изменился в новых версиях 1С | Низкая | HBK — стабильный zip+HTML уже 10+ лет |
| Java + BSL LS требует много RAM | Средняя | Alpine + JRE 17 ~300MB; если критично — отключить sidecar |
| 50 шаблонов недостаточно | Средняя | Начать с 50, расширять по фидбеку; добавить API для пользовательских шаблонов в v0.5 |
| BERTA плохо эмбеддит русскую документацию 1С | Низкая | BERTA обучена на русском; BM25 подстрахует через гибридный поиск |

**Открытые вопросы:**
- Нужно ли копировать весь `bin/` на VPS или только `1cv8_ru.hbk`? (Решение: только ru.hbk — ~2 MB)
- Какую версию Java использовать? (Решение: eclipse-temurin:17-jre-alpine — компактная, LTS)

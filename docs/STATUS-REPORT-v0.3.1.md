**Типичный сценарий аналитика:**
1. `1c_metadata_search` → найти нужный объект
2. `1c_odata_query` / `1c_register_balances` → получить данные из живой базы
3. Анализ и выводы

#### Для разработчика 1С

| Инструмент | Что делает | Пример |
|---|---|---|
| Все 7 выше | + | + |
| `1c_code_search` | Поиск по BSL-коду: процедуры, функции | «ОбработкаПроведения» в конкретном документе |

**Типичный сценарий разработчика:**
1. `1c_metadata_search` → найти объект

3. `1c_subsystems(find)` → понять бизнес-роль объекта

5. `1c_code_search` → найти существующий код
6. Написать код с полным контекстом

## 4 внешних MCP-сервера от comol/DISTAR — готовы к запуску

Лицензионные ключи получены из дистрибутива MCP Distr, файл `.env.comol` создан.

| Сервер | Порт | Для кого | Что даёт | Ключ |
|---|---|---|---|---|
| `1c_help_mcp` | 8003 | Разработчик | Справка по платформе 1С:Предприятие 8.3 | есть |
| `template-search-mcp` | 8004 | Разработчик | Шаблоны кода 1С + веб-редактор (`/extend/`) | есть |
| `1c_syntaxcheck_mcp` | 8002 | Разработчик | Проверка синтаксиса BSL (BSL Language Server) | есть |
| `mcp_ssl_server` | 8008 | Разработчик | Справка по БСП v3.1.11 | есть |

**Запуск одной командой:**
```bash
docker compose -f docker-compose.comol.yml --env-file .env.comol up -d
```

**Важно при первом запуске:**
- Help и SSL серверы скачивают модели и индексируют данные (до нескольких часов)
- Каждый сервер требует ~10 ГБ дискового пространства
- Для ускорения индексации можно подключить LM Studio с моделью Qwen3 Embedding 4B/8B
- Прогресс можно отслеживать через `docker logs`

## Использовали ли мы дистрибутивы comol (MCP Distr)?

**Нет, мы их НЕ использовали.** Образы comol прописаны в `docker-compose.comol.yml`, но ни разу не запускались. Из дистрибутива взяты только лицензионные ключи.

Всё, что сейчас работает — **написано нами с нуля:**

| Что написали сами | Аналог в comol |
|---|---|
| `1c_metadata_search` (гибридный поиск BERTA+BM25) | `1c_code_metadata_mcp` (ChromaDB) — дублирует |
| `1c_code_search` (поиск по BSL-коду) | `1c_code_metadata_mcp` — дублирует |
| `1c_dependencies` (граф из Qdrant, без Neo4j) | `1c_graph_metadata` (Neo4j + 2GB + OpenAI) — тяжелее |
| `1c_subsystems` (навигация по подсистемам) | **Нет аналога ни у кого** |
| `1c_odata_query` + `1c_register_balances` + `1c_register_movements` | **Нет аналога ни у кого** |
| Скилл `1c-analyst` | **Нет аналога ни у кого** |

**Уникальные фичи (только у нас):**
- OData-интеграция (3 инструмента для живых данных из 1С)
- Навигация по подсистемам
- Аналитический скилл с 50+ паттернами анализа

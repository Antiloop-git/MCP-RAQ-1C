# Настройка OData для MCP-сервера

## Шаг 1. Публикация базы на веб-сервере

1. Открыть базу в **Конфигураторе** 1С
2. Меню **Администрирование → Публикация на веб-сервере...**
3. Заполнить:
   - **Имя:** `erp` (или любое другое)
   - Галочка **«Публиковать стандартный интерфейс OData»**
4. Нажать **Опубликовать**

Результат: `http://<сервер>/erp/odata/standard.odata/`

## Шаг 2. Включить состав OData

Выполнить в **консоли кода 1С** (или через внешнюю обработку):

```bsl
// === Регистры накопления (для анализа остатков) ===
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.РегистрыНакопления.ДвижениеТМЦ, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.РегистрыНакопления.УчетПартий, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.РегистрыНакопления.ПартииНоменклатуры, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.РегистрыНакопления.Реализация, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.РегистрыНакопления.Отгрузка, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.РегистрыНакопления.УчетВзаиморасчетов, Истина);

// === Справочники ===
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.Справочники.Номенклатура, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.Справочники.Склады, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.Справочники.Организации, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.Справочники.Контрагенты, Истина);

// === Документы (основные товарные) ===
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.Документы.РеализацияТоваровУслуг, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.Документы.ПоступлениеТоваровУслуг, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.Документы.ПеремещениеТоваров, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.Документы.СписаниеТоваров, Истина);
УстановитьСоставСтандартногоИнтерфейсаOData(Метаданные.Документы.ИнвентаризацияТоваров, Истина);

Сообщить("Состав OData настроен!");
```

> **Важно:** Имена документов могут отличаться в вашей конфигурации. Проверьте точные имена через `1c_metadata_search`.

## Шаг 3. Создать пользователя для OData

В 1С: **Администрирование → Пользователи** — создайте пользователя (например `odata_reader`) с правами:
- Чтение справочников
- Чтение документов
- Чтение регистров накопления

## Шаг 4. Проверить доступ

```bash
# Проверка подключения
curl -u "odata_reader:пароль" \
  "http://сервер/erp/odata/standard.odata/?$format=json"

# Получить справочник Номенклатура (первые 5)
curl -u "odata_reader:пароль" \
  "http://сервер/erp/odata/standard.odata/Catalog_Номенклатура?$format=json&$top=5"

# Остатки ДвижениеТМЦ с отрицательным количеством
curl -u "odata_reader:пароль" \
  "http://сервер/erp/odata/standard.odata/AccumulationRegister_ДвижениеТМЦ_Balance?$format=json&$filter=КоличествоBalance lt 0&$top=10"
```

## Шаг 5. Настроить MCP-сервер

Добавить в `.env` файл проекта:

```env
ODATA_URL=http://сервер/erp/odata/standard.odata
ODATA_USER=odata_reader
ODATA_PASSWORD=пароль
```

Перезапустить:

```bash
docker compose up mcp-server --build -d
```

После этого в MCP-сервере появятся 3 новых инструмента:
- `1c_odata_query` — универсальный запрос
- `1c_register_balances` — остатки регистров
- `1c_register_movements` — движения регистров

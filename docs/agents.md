# MCP RAQ 1C — Инструкция для LLM-агентов

Этот MCP-сервер предоставляет доступ к метаданным конфигурации 1С.
Используй его, чтобы узнать структуру объектов перед написанием кода или запросов на языке 1С.

## Доступные инструменты

### 1c_metadata_search

Поиск объектов метаданных по текстовому запросу. Гибридный поиск: по техническому имени, синониму и полному описанию.

**Параметры:**
- `query` (string, обязательный) — поисковый запрос на русском или техническое имя
- `object_type` (string, необязательный) — фильтр: `Catalog`, `Document`, `AccumulationRegister`, `InformationRegister`, `AccountingRegister`, `Enum`
- `limit` (number, необязательный, по умолчанию 10, макс 50)

**Возвращает** компактный список: имя, синоним, тип, score. Не содержит полного описания — для этого используй `1c_metadata_details`.

**Примеры запросов:**
```
1c_metadata_search(query: "заказ клиента")
1c_metadata_search(query: "остатки товаров", object_type: "AccumulationRegister")
1c_metadata_search(query: "номенклатура", object_type: "Catalog")
1c_metadata_search(query: "EDIПровайдеры")
```

### 1c_metadata_details

Полное описание объекта по техническому имени. Возвращает реквизиты, табличные части, измерения, ресурсы, движения регистров.

**Параметры:**
- `name` (string, обязательный) — техническое имя объекта (например, `SS_ЗаказКлиента`)

Если объект не найден, сервер подскажет ближайшие совпадения.

**Примеры:**
```
1c_metadata_details(name: "SS_ЗаказКлиента")
1c_metadata_details(name: "Номенклатура")
```

### 1c_metadata_types

Статистика по типам объектов в конфигурации: сколько справочников, документов, регистров и т.д.

**Параметры:** нет.

## Рекомендуемый workflow

1. **Поиск** — вызови `1c_metadata_search` с описанием того, что ищешь
2. **Выбор** — из результатов выбери подходящий объект по имени и синониму
3. **Детали** — вызови `1c_metadata_details` с техническим именем объекта
4. **Код** — используй полученную структуру для написания запроса или кода 1С

**Пример:** пользователь просит "Напиши запрос для получения остатков товаров"
```
Шаг 1: 1c_metadata_search(query: "остатки товаров", object_type: "AccumulationRegister")
        -> Находим: SS_ОстаткиТоваровНаСкладах (Остатки товаров на складах)

Шаг 2: 1c_metadata_details(name: "SS_ОстаткиТоваровНаСкладах")
        -> Получаем: измерения (Номенклатура, Склад), ресурсы (Количество, Сумма)

Шаг 3: Пишем корректный запрос:
        ВЫБРАТЬ Номенклатура, Склад, КоличествоОстаток, СуммаОстаток
        ИЗ РегистрНакопления.SS_ОстаткиТоваровНаСкладах.Остатки
```

## Советы по формированию запросов

- Ищи по **синониму** на русском: "контрагенты", "заказ клиента", "остатки товаров"
- Ищи по **техническому имени** если знаешь: "SS_ЗаказКлиента", "EDIПровайдеры"
- Используй **фильтр по типу** для уточнения: `object_type: "Document"` если ищешь документ
- Технические имена в конфигурации могут содержать префиксы (например, `SS_`, `Смолл_`)
